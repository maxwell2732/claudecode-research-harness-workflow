# Frontend Design Summary

Design guidelines for enhancing UI distinctiveness and aesthetic completeness.
Apply only when explicitly requested.

## Design Thinking (Decide First)
- Purpose: Whose problem does this UI solve?
- Tone: Choose one bold direction (e.g.: minimal / brutalist / editorial / playful)
- Constraints: Technical constraints and a11y
- Differentiation: Decide the "one memorable thing" about this UI

## Implementation Quality Standards
- Production-grade and actually works
- Visually striking with a consistent direction
- Attention to detail (spacing, hierarchy, states)

## Design Guidelines
- Typography: A display font that catches the eye, paired with readable body text
- Color: A clear primary color and a sharp accent
- Motion: Focus on one large impactful effect (don't scatter)
- Space: Intentionally choose either generous whitespace or high density
- Background: Atmospheric layers or textures if needed

## Notes
- Direction requires "strength of intent". It's not about flashiness.
- Implementation complexity should match the design direction.
